# Техническое задание: Text Complexity Analyzer

## 1. Цель проекта
Создание веб-сервиса для автоматической оценки сложности текста. Система позволяет пользователям загружать тексты, классифицировать их по категориям и получать автоматический анализ сложности (количество слов, предложений, уникальных слов) с визуализацией данных.

## 2. Роли пользователей
*   **Гость**: Просмотр публичных текстов и их списков.
*   **Авторизованный пользователь**: Загрузка собственных текстов, редактирование, просмотр детальной аналитики.
*   **Администратор**: Управление категориями, модерация текстов.

## 3. Модели данных
### 1. Category (Категория)
Группировка текстов по жанрам или типам (научный, художественный, новости).
*   `name`: CharField (Название)
*   `description`: TextField (Описание)

### 2. TextDocument (Текстовый документ)
Основная сущность, содержащая анализируемый текст.
*   `title`: CharField (Заголовок)
*   `content`: TextField (Текст)
*   `created_at`: DateTimeField (Дата создания)
*   `category`: ForeignKey -> Category (Категория текста)

### 3. Tag (Тег)
Метки для дополнительной классификации (для связи ManyToMany).
*   `name`: CharField (Название тега)
*   `documents`: ManyToManyField -> TextDocument (Связь с документами)

## 4. Ключевой функционал (User Stories)
*   **Загрузка текста**: Пользователь заполняет форму (Заголовок, Категория, Текст, Теги), система сохраняет документ.
*   **Анализ**: При просмотре документа система отображает статистику (число слов, частотный словарь).
*   **Визуализация**: На странице детального просмотра отображается диаграмма распределения длин слов или топ-10 частых слов.
*   **Фильтрация**: Список текстов можно фильтровать по категориям.

## 5. Внешние интеграции / Аналитика
*   **Pandas**: Обработка текста, подсчет частотности слов.
*   **Matplotlib**: Генерация графиков статистики текста.
